# AGENTS.md

High-signal notes for OpenCode agents working in this repo.

## Repo shape and entrypoints
- This repo is a Bash-based Distrobox setup; core entrypoints are executable scripts.
- `install` copies `user/local/share/towel/*` into `$HOME/.local/share/towel`, copies `user/config/{containers,distrobox}` into `$HOME/.config`, and symlinks `$HOME/.local/bin/towel`.
- Runtime CLI entrypoint is `user/local/share/towel/bin/towel`; each subcommand is its own executable under `user/local/share/towel/bin/towel-*`, with shared helpers in `user/local/share/towel/bin/towel-common`.

## Behavior quirks worth preserving
- `install` intentionally hardens shell behavior by enabling builtin commands and unaliasing names (`enable` + `unalias`), and uses absolute paths like `/usr/bin/cp`; keep this style in installer changes.
- `install` uses guarded cleanup on failure: it only removes install targets created by the current run, and only when paths exactly match expected `$HOME` targets.
- `install` refuses non-interactive runs unless `--yes` is provided.
- `install` PATH update is conservative: it preserves a literal single-quoted export line, skips if rc already contains a `$HOME/.local/bin` entry, and prompts interactively when runtime PATH contains it but rc evidence is ambiguous.
- `.distrobox-pin` is sourced by `install`; keep `DISTROBOX_VERSION/URL/SHA256` consistent when changing pins.
- Export wrappers are generated by towel itself (shared helper logic in `bin/towel-common`), not by `distrobox-export` patching. Bin wrappers call `towel exec <cmd> "$@"` and may self-remove on failure; desktop wrappers are copied to `~/.local/share/applications/towel-*.desktop` with rewritten `Exec=`.
- `towel export <pkg...>` is implemented and runs export logic inside the towel container (via host/container trampoline) using `dnf repoquery` to discover `/usr/bin/*` and `/usr/share/applications/*.desktop` candidates.

## CI and release expectations
- CI runs `shellcheck -x install user/local/share/towel/bin/towel user/local/share/towel/bin/towel-*` and `shfmt -d` on the same paths; these are the fast verification steps to mirror locally.
- CI has an install smoke test that runs `./install -y` inside an Ubuntu 22.04 container as an unprivileged user; keep installer behavior non-interactive-safe.
- Releases are driven by `npx semantic-release` (Node 20, `npm ci`), with version written to `VERSION` and a source tarball + sha256 produced from repo root; avoid edits that break `semantic-release --dry-run --branches main`.
- PR titles are linted against conventional commit types (feat/fix/chore/docs/refactor/test/ci/build/perf/revert/style).

## Verification commands (focused)
- Lint executable scripts directly (files do not use `.sh` extensions):
  - `shellcheck -x install user/local/share/towel/bin/towel user/local/share/towel/bin/towel-*`
- Format only shell scripts, not the whole repo:
  - `shfmt -w install user/local/share/towel/bin/towel user/local/share/towel/bin/towel-*`

## Style conventions observed in code
- Tabs are used for indentation in shell scripts.
- Existing files mix `#!/usr/bin/env bash` and `#!/bin/bash`; prefer `#!/usr/bin/env bash` for new/updated executable scripts unless matching an existing file is required.
- Quote variable expansions consistently; scripts rely on host/container boundary checks via `CONTAINER_ID` and should keep stderr messaging for misuse paths.
